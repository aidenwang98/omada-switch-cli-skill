## ipv6 pim anycast-rp

### Purpose
Anycast RP is an extension mechanism of the PIM-SM protocol. By configuring the same anycast address for multiple physical RPs, it enables multicast sources to register with the nearest RP and receivers to join the nearest RP, addressing issues in the traditional single-RP model such as traffic concentration, slow fault convergence, and suboptimal paths. During configuration, it is recommended to configure the anycast-rp and member RP addresses as loopback addresses. Each anycast-rp member device must be configured with all member RPs. During deconfiguration, you can delete the entire anycast-rp configuration or remove only a specific member RP configuration.

### Command Mode
Global Configuration Mode

### Syntax
```text
ipv6 pim anycast-rp anycast_rp_addr member_addr [ vrf vrfname ]
no ipv6 pim anycast-rp anycast_rp_addr [ member_addr ] [ vrf vrfname ]
```

### Parameters
```text
anycast_rp_addr : The anycast-rp address, representing a set of RPs.
member_addr: A member rp address within the RP set.
vrfname: Virtual Routing and Forwarding instance name.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Assume Switch A and Switch B form an anycast-RP. The anycast-RP address for both Switch A and Switch B is 1:1::1:1. Switch A
s member-RP is 2:2::2:2, and Switch B
s member-RP is 3:3::3:3.
Taking Switch A configuration as an example, both local and remote member RPs must be configured:
Switch(config)# ipv6 pim anycast-rp 1:1::1:1 2:2::2:2
Switch(config)# ipv6 pim anycast-rp 1:1::1:1 3:3::3:3
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## ipv6 pim bsr-candidate

### Purpose
This command is used to configure a candidate BSR on the interface.

### Command Mode
Global Configuration Mode

### Syntax
```text
ipv6 pim bsr-candidate interface intf_type intf_id [ hash-mask-length ] [ priority ] [ interval ] [ vrf vrfname ]
no ipv6 pim bsr-candidate [ vrf vrfname ]
```

### Parameters
```text
intf_type: Interface type used for C-BSR.
intf_id: Interface ID used for C-BSR.
hash-mask-length: Hash mask length used to map multicast group addresses to RPs. Default is 30, range 0 - 32.
priority: C-BSR election priority, where a higher value wins. Default is 0, range 0-255.
interval: Interval for sending PIM BSR messages, in seconds. Default is 60 seconds, range 1 - 16383.
vrfname: Virtual Routing and Forwarding instance name.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Switch(config)# ipv6 pim bsr-candidate int loopback 1 vrf test
Switch(config)# no ipv6 pim bsr-candidate vrf test
```

- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.

## ipv6 pim register-source

### Purpose
This command configures the IP header source address of register packets sent by the Designated Router (DR). You can configure an IP address or an interface, using the interface s IP address. The configured source address should be unicast-reachable to the RP to ensure proper PIM-SM registration and registration-stop processes. By default, the source address is the RPF interface address pointing to the multicast source.

### Command Mode
Global Configuration Mode

### Syntax
```text
Ipv6 pim register-source {ip_addr | interface intf_type intf_id} [ vrf vrfname ]
no ipv6 pim register-source [ vrf vrfname ]
```

### Parameters
```text
ip_addr: IP address to be used as the source address for register messages.
intf_type : Interface type to be used for the register message source address.
intf_id : Interface ID to be used for the register message source address.
vrfname: Virtual Routing and Forwarding instance name.
```

### Default
By default, the source address is the RPF interface address pointing to the multicast source.

### Example
```text
Switch(config)# ipv6 pim register-source 2001:1::101
Switch(config)# ipv6 pim register-source interface loopback 1
Switch(config)# no ipv6 pim register-source
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.
