## Chapter Notes

Official documentation states: Tunnel is an encapsulation technique that uses one network protocol to transport another network protocol. It encapsulates data packets generated by another protocol within its own packets, enabling the transmission of IPv6 packets over an IPv4 network and facilitating interconnection between IPv6 networks. A tunnel represents a virtual point-to-point connection where it provides a channel for the transmission of encapsulated data packets, allowing for encapsulation and decapsulation of data packets at both ends of the tunnel. EoGRE (Ethernet over GRE) is a tunneling technology used to encapsulate Ethernet frames for transmission over an IP network. By carrying Ethernet traffic from multiple VLANs over an EoGRE tunnel, it enables Layer 2 network interconnection across different geographical locations.

- This chapter is marked `Only for Certain Devices`; treat all configuration commands as model-dependent.

## interface tunnel

### Purpose
This command is used to create a Tunnel entry and enters Tunnel Interface Configuration Mode. Its no command deletes the configuration information and removes the tunnel.

### Command Mode
Global Configuration Mode

### Syntax
```text
interface tunnel tunnel-number
no interface tunnel tunnel-number
```

### Parameters
```text
tunnel-number: Tunnel entry index value, range 1
1000. Only one value can be entered per command.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Create a tunnel with an index value of 10 and enter Tunnel Interface Configuration Mode:
Switch(config)# interface tunnel 10
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## tunnel mode

### Purpose
This command is used to specify the tunnel mode. The tunnel will encapsulate different protocol headers for passenger messages according to different tunnel types. To delete the type of the tunnel interface, please use the no tunnel mode command.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
tunnel mode { gre { eth-ipv4 | ipv4-ipv4 | ipv6-ipv4} | ipv6ip}
no tunnel mode
```

### Parameters
```text
gre eth-ipv4
Specifies Ethernet over IPv4 tunnel mode, transmitting Ethernet frames over IPv4.
gre ipv4-ipv4
Specifies GRE IPv4 over IPv4 tunnel mode, transmitting IPv4 packets over IPv4.
gre ipv6-ipv4
Specifies GRE IPv6 over IPv4 tunnel mode, transmitting IPv6 packets over IPv4.
ipv6ip
Specifies IPv6 over IPv4 tunnel mode, transmitting IPv6 packets over IPv4.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure Tunnel 10 to use EoGRE mode:
Switch(config)# interface tunnel 10
Switch(config-tunnel)# tunnel mode gre eth-ipv4
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## tunnel ttl

### Purpose
This command is used to configure the TTL parameter for the tunnel. TTL (Time to Live) is a field in network protocols used to control the lifetime of a packet within a network. In an EoGRE tunnel, the TTL value determines the maximum number of hops the encapsulated outer IP packet is allowed to traverse in the network. This helps prevent packets from looping indefinitely and limits the propagation distance of packets across wide area networks.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
tunnel ttl ttl-value
no tunnel ttl
```

### Parameters
```text
ttl-value
The Time to Live value, range 1
255. Default is 255.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure the TTL for Tunnel 10 to 254:
Switch(config)# interface tunnel 10
Switch(config-tunnel)# tunnel ttl 254
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## tunnel ip address

### Purpose
This command is used to specify the interface IP address of the tunnel. GRE IPv4-IPv4 tunnels need to be configured with an IPv4 address, and IPv6IP and GRE IPV6-IPv4 tunnels need to be configured with an IPv6 address. To delete the IP address of the tunnel, please use the no ip address command.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
ip address ipv4-address / ipv6 address ipv6-address
no ip address / no ipv6 address
```

### Parameters
```text
Ipv4-address: IPv4 address of the tunnel interface.
ipv6-address: IPv6 address of the tunnel interface.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Specify the interface IP address of the GRE IPv4-IPv4 tunnel as 10.1.1.1:
Switch(config)#interface tunnel 1
Switch(config-tunnel) ip address 10.1.1.1
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## tunnel source

### Purpose
This command is used to configure the source IP address for the tunnel. The tunnel source can be specified in three modes: IP address, Layer 3 interface, or VlLANIf.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
tunnel source {ip address ip-address | interface {gigabitEthernet | hundred-gigabitEthernet | port-channel | ten-gigabitEthernet | twentyFive-gigabitEthernet | two-gigabitEthernet} port-number | vlan-if vlanId}
no tunnel source
```

### Parameters
```text
ip-address
Source IP address.
port-number
Port number, e.g., gigabitEthernet 1/0/1.
vlanId
Index value of the Layer 3 VLAN interface.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure the source IP for Tunnel 10 as 192.168.10.1:
Switch(config)# interface tunnel 10
Switch(config-tunnel)# tunnel source ip address 192.168.10.1
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## keepalive period retry-times

### Purpose
This command is used to configure the keepalive parameters for the tunnel, which is a mechanism used to detect and maintain the tunnel connection state.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
keepalive period period retry-times times
```

### Parameters
```text
period
Keepalive packet transmission interval, in seconds. Range 1
6000. Default is 5 seconds.
times
Retry count. Range 1
255. Default is 3.
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure keepalive for Tunnel 10: send a probe every 10 seconds; consider the tunnel down after 5 consecutive timeouts:
Switch(config)# interface tunnel 10
Switch(config-tunnel)# keepalive period 10 retry-times 5
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## tunnel vlan

### Purpose
This command is used to configure the access-side VLAN(s) for the EoGRE tunnel. A single EoGRE tunnel can correspond to one VLAN or multiple VLANs. The VLAN(s) must already exist before configuration.

### Command Mode
Tunnel Configuration Mode

### Syntax
```text
tunnel vlan { vlanId | vlan-range}
no tunnel vlan
```

### Parameters
```text
vlanId
A single VLAN index value, range 1
4094.
vlan-range
A VLAN range, supporting the following formats:
Continuous range: 1-100
Multiple ranges: 1-10,50-100
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure VLAN 100 as the access-side VLAN for EoGRE Tunnel 10:
Switch(config)# interface tunnel 10
Switch(config-tunnel)# tunnel vlan 100
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

## ip route tunnel

### Purpose
This command is used to configure a static route for IPv4 packets to point to a tunnel, so that IPv4 packets can enter the tunnel (including only GRE IPv4-IPv4 type tunnels). To delete the static route configuration, please use the no command.

### Command Mode
Global Configuration Mode

### Syntax
```text
ip route ip-address mask tunnel tunnel-number
no ip route ip-address mask tunnel tunnel-number
```

### Parameters
```text
ip-address: IP address of the static route.
mask: Mask of the static route.
tunnel-number: Tunnel interface to which the static route points
```

### Default
Not explicitly specified in the official documentation.

### Example
```text
Configure the IPv4 static route to point to tunnel interface 1:
Switch(config)#ip route 192.168.2.0 255.255.255.0 tunnel 1
```

### Notes
- Permission requirement: Only Admin, Operator and Power User level users have access to these commands.
- Availability: Only for certain devices according to the official chapter title.

### Parameters
```text

tunnel-number: Tunnel interface to which the static route points
```

### Example
